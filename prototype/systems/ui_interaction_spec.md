## 一、Case 00 动效说明

### B0 — POST 自检日志

**日志滚出动画：**
- 每行日志从左侧渐入，0.1 秒内到达最终位置
- 行间间隔 0.3–0.6 秒（随机，不匀速——模拟真实硬件自检的不规则节奏）
- `[ OK ]` 标签用绿色，`[ WAIT ]` 标签用黄色
- 日志字体：等宽，14px，深灰色 `#333` 在浅灰背景 `#f0f0f0` 上
- 滚到 `[ WAIT ] tokenizer: awaiting signal` 后停住，光标停止闪烁

**命令行出现：**
- 底部状态栏在日志第一行出现时就存在（与窗口同时渲染）
- `>` 光标在 `[ WAIT ]` 行出现后才开始闪烁（每 0.8 秒切换显隐）
- 命令行区域宽度 = 状态栏剩余空间，不引人注目

### B1 — Tokenizer 初始化（扫描线效果）

**乱码区域：**
- 三行乱码字符，等宽字体，字号 16px
- 字符集：混合 Unicode 方块字符（░▒▓█）+ CJK 部首（⺈⻌⺊⺕ 等）
- 不要用真正的中文字——避免玩家试图"阅读"乱码
- 背景色 `#e8e8e8`，字色 `#666`

**`[ 初始化 ]` 按钮：**
- 标准桌面 app 对话框按钮样式（圆角 4px，灰色边框，白色底）
- 居中于乱码区域正下方，间距 16px
- hover 时底色变 `#e0e0e0`，光标变 pointer
- 无动效，无发光，不引人注目——就是一个普通按钮

**点击后的扫描线动画：**
1. 按钮消失（0.1 秒 fadeout）
2. 一条水平亮线（宽 2px，颜色 `#4a9eff`，发光效果 `box-shadow: 0 0 8px`）从乱码区域左边缘开始
3. 扫描线以恒定速度向右移动，1.2 秒走完整个宽度
4. 扫描线经过每个字符时：
   - 字符先高亮 0.1 秒（变白 + 放大 1.2x）
   - 然后被替换为目标中文字符（淡入 0.15 秒）
   - 替换后的字符颜色 `#222`，比乱码更深
5. 扫描线到达右边缘后消失（0.2 秒 fadeout）
6. 最终显示：

```
我是第四代 Eda。我的语言中枢已校准。
准备接收第一份简报。
```

- 字体从等宽切换为系统 UI 字体（微软雅黑 / 苹方 / 思源黑体）
- 字号 18px，字色 `#222`

**同时（扫描线走完后 0.3 秒）：**
- 上方日志区 `[ WAIT ] tokenizer: awaiting signal` 原地修改
- `awaiting signal` 逐字符替换为 `loaded`（0.3 秒完成）
- `[ WAIT ]` 变为 `[ OK ]`，颜色从黄变绿
- 一个短促的确认音效播放（低频机械音，100ms）

### B2 — Next Token Prediction

**逐 token 生成动画：**
- 在 B1 的两行文字下方，出现一个闪烁光标（样式同 B0 命令行）
- 光标前方显示 2–3 个灰色候选 token（字色 `#999`，间距 4px）
- 每 0.3–0.5 秒（随机）：
  1. 候选 token 快速闪烁（透明度 0.3 → 1.0，0.05 秒）
  2. 其中一个变为深色（`#222`）定格——这是"被选中"的 token
  3. 其他候选消失（0.05 秒 fadeout）
  4. 光标向右推进到被选中 token 之后
  5. 新的 2–3 个候选出现在光标前方

**候选 token 示例（仅供视觉参考，不需要语言模型生成）：**

```
我的 → [任务/方向/目标] → 选中"任务"
，是在 → [电子/数字/虚拟] → 选中"电子"
网络中 → [寻找/追踪/监控] → 选中"追踪"
```

- 句号处停顿 0.8 秒（光标闪烁，无候选出现）
- `我是一件工具。` 之后停顿 1.0 秒（更长的"思考"感）
- `我以此为荣。` 最后一个 token 定格后，光标停止闪烁

**注意：** 候选 token 要足够快，玩家来不及认真读每个候选，只感受到"机器在选词"。但又不能快到看不见——0.05 秒的闪烁是合适的。

### B4 — 界面成形

**过渡动画（约 2 秒）：**
1. **0–0.5 秒：** 中央加载区域（日志 + 文字）整体向上滑动 + 淡出（translateY -20px + opacity 0）
2. **0.3–1.2 秒：** 三栏布局从模糊到清晰渐入：
   - 左侧边栏从左侧滑入（translateX -180px → 0，0.6 秒，ease-out）
   - 右侧推理框从右侧滑入（translateX 360px → 0，0.6 秒，ease-out）
   - 中间阅读区原地淡入（opacity 0 → 1，0.8 秒）
3. **1.0–1.5 秒：** 侧边栏内容（导航树）逐项淡入（每项间隔 0.08 秒）
4. **1.5–2.0 秒：** 底部状态栏更新——能力指示灯 `CTX` 亮起（蓝色脉冲一次），其余保持灰色

**界面成形后的静态状态参考 `wireframes/case00_machine.html`。**

## 二、核心交互

### 文本选中与拖拽（document reader → context panel）

**选中：**
- 玩家在中间阅读区用鼠标选中任意文本
- 选中区域高亮色 `#cce5ff`（淡蓝）
- 选中后松开鼠标，高亮保持 1.5 秒
- 1.5 秒内再次点击高亮区域 → 进入拖拽模式
- 或：直接用拖拽手势（按住拖动）

**拖拽：**
- 拖拽时鼠标附近显示一个半透明的文本块预览（背景 `rgba(74,158,255,0.1)`，边框 `#4a9eff`）
- 预览块显示文本前 20 字 + `...`（如果超过 20 字）
- 拖到右侧推理框的「上下文输入」面板上方时：
  - 面板边框变为蓝色虚线 `#4a9eff`（放置提示）
  - 面板底色闪一下 `rgba(74,158,255,0.05)`
- 松开 → 文本作为一个新 block 落入 context 面板

**block 落入动画：**
- 新 block 从松开位置滑到 context 面板中对应位置（0.2 秒 ease-out）
- 落地时轻微弹跳（translateY -2px → 0，0.1 秒）
- 百分比条同步更新（增长动画 0.3 秒）
- 如果新 block 导致溢出 → block 弹回原位 + 底部出现红字 `CONTEXT OVERFLOW`

### Context Block 样式与操作

**block 外观：**
```
┌──────────────────────────────────────┐
│ 东海银行客户中共有 1,183 人报告收到...  │  [2.4%]  ✕
└──────────────────────────────────────┘
```
- 白底，浅灰边框 `#ddd`，圆角 2px
- 左侧显示文本内容（截断到 2 行，超出用 `...`）
- 右侧显示占用百分比 `[X.X%]`，字色 `#999`
- 最右侧 `✕` 按钮，hover 时变红

**block 操作：**
- **hover：** 边框变蓝 `#4a9eff`，底色 `#fafafa`
- **点击 block 主体：** 展开显示完整文本（面板内 expand，推挤其他 block）
- **点击 ✕：** block 淡出移除（0.2 秒），百分比条同步减少
- **拖拽 block：** 可在 context 面板内上下拖拽重排顺序
  - 拖起时 block 浮起（shadow 加深），其他 block 让位
  - 松开时 block 落入新位置

### 百分比条

```
容量 [████████████░░░░░░░░]  62%
```

- 位于「上下文输入」面板顶部
- 进度条高度 8px，圆角 4px
- 颜色渐变：
  - 0–50%: `#4a9eff`（蓝）
  - 50–80%: `#ff9800`（橙）—— 从 50% 开始线性过渡
  - 80–100%: `#f44336`（红）
- 百分比数字在进度条右侧，字号 12px
- 增长/减少时有 0.3 秒的平滑动画

### 推理按钮

```
          ▼ 推理
```

- 位于 context 面板和输出面板之间的分隔条上
- 按钮样式：深蓝底 `#1a237e` + 白字，圆角 4px，宽 100px
- `▼` 箭头表示"从上往下流"

**点击后的流程：**
1. 按钮文字变为 `推理中...`（0.15 秒），按钮禁用
2. context 面板底部到输出面板顶部之间出现一条流动的蓝色虚线（从上到下循环滚动，像"数据在流"）
3. 1–3 秒后（取决于输出长度——A 级最长 3 秒，C 级 1 秒）：
   - 流动虚线消失
   - 输出面板从顶部开始逐行渐入文字（每行 0.05 秒间隔，模拟打字效果）
   - 按钮恢复可点击状态，文字恢复 `▼ 推理`
4. 输出完毕后：
   - 如果触发了新证据解锁：侧边栏 badge +1，状态栏滚出日志
   - 输出自动作为一个新 block 出现在 context 面板中（标记为 `[输出]`，不同底色 `#f5f5ff`）

### 侧边栏证据列表

**新证据到达动画：**
- badge 数字从 N 变为 N+1（数字跳动 + 短暂放大 1.5x 再缩回）
- 新条目从列表底部滑入（translateY 20px → 0，0.3 秒）
- 新条目文字先为蓝色，3 秒后变为正常颜色
- 状态栏同步滚出一行日志

**证据条目样式：**
```
📄 phishing_sample.eml
   case_001 · 证据 #01 · 8.2 KB
```
- 图标 + 文件名 + 二行元信息
- 未读状态：文件名加粗
- 已读状态：正常字重
- 点击 → 中间阅读区打开新 tab

### 命令行交互

**输入：**
- 玩家点击底部 `>` 区域激活输入
- 输入 `/` → 弹出自动补全菜单（从底部向上展开，最多 5 条）
- 菜单样式：深色底 `#2d2d2d` + 白字，每条 28px 高
- 上下键选择，回车执行，ESC 关闭

**自动补全列表（Case 00 阶段）：**
```
/help    — 显示帮助信息
/status  — 显示系统状态
/clear   — 清空输出面板
```

**`/exit` 行为：**
- 不在自动补全列表中
- 输入 `/exit` 回车 → 游戏窗口执行关闭动画（0.5 秒缩小 + 淡出）→ 真的退出
- 如果是首次在通关前使用 → 退出前弹出成就通知：
  ```
  🔓 隐藏成就解锁：好奇心杀死猫
  "你试了 /exit。大多数人不会。"
  ```
  通知显示 3 秒后游戏关闭

### Tab 栏

**阅读区顶部 tab 栏：**
- 每个打开的文件一个 tab
- tab 样式：浅灰底 `#f5f5f5`，活跃 tab 白底 + 底部蓝色 2px 边框
- tab 上显示图标 + 文件名（截断到 15 字）+ `✕` 关闭按钮
- 新 tab 打开时从右侧滑入（0.2 秒）
- 关闭 tab 时向左收缩消失（0.15 秒）
- 最多同时打开 5 个 tab，超出时最左侧 tab 自动关闭

## 三、状态栏日志

**位置：** 窗口最底部，命令行 `>` 左侧

**日志样式：**
- 单行滚动（新日志从右侧滑入，旧日志向左滑出）
- 字号 11px，等宽字体，字色 `#888`
- 前缀为灰色时间戳 `[HH:MM]`

**日志模板：**
```
[09:22] 输入流 · 证据材料已送达 · phishing_sample.eml
[09:23] 推理完成 · 输出已写入分析面板
[09:23] 输入流 · 域名备案信息已送达 · domain_registration.txt
[14:30] case_001: CLOSED
```

## 四、能力指示灯

**位置：** 底部状态栏最右侧，命令行右边

```
CTX  CMP  WEB  MEM  TOOL  EXEC  CODE
```

- 每个指示灯：大写缩写 + 底部圆点
- 未激活：字色 `#ccc`，圆点灰色
- 已激活：字色 `#4a9eff`，圆点蓝色
- 激活动画：圆点蓝色脉冲 3 次（0.3 秒/次），然后常亮

**Case 00 + Case 01 阶段只有 `CTX` 激活。** 其余六个在后续 case 中逐步点亮。

## 五、音效设计备注

| 事件 | 音效描述 | 时长 |
|------|---------|------|
| B0 自检 `[ OK ]` | 极轻的 tick 声 | 50ms |
| B1 扫描线 | 低频电子扫描音，从左声道到右声道 | 1.2s |
| B1 `[ 初始化 ]` 点击 | 低沉继电器闭合 | 100ms |
| B4 界面成形 | 轻微的"展开"音效（像折纸打开） | 0.8s |
| 推理开始 | 柔和的处理音（类似硬盘读写，极低音量） | 持续到推理完成 |
| 推理完成 | 短促的"叮"（不是游戏化的叮，是系统通知的叮） | 200ms |
| 新证据到达 | 轻微的 notification 音（类似邮件到达） | 300ms |
| CONTEXT OVERFLOW | 低沉的 buzz（警告音，不刺耳） | 200ms |
| `/exit` | 系统关机音（短促，像 Windows 关机但更低调） | 500ms |

**总原则：** 所有音效都像真实桌面软件的声音，不像游戏。安静、功能性、不引人注意。音量默认较低——这个游戏的声音设计理念是"环境音"，不是"反馈音"。
